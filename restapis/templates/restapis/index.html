Naive TODO-App APIs

Following Page describes how to use APIs.

A total of Three APIs are there as required by the task. Since Attributes didn't mentioned any requirement for USER ASSOCIATION, APIs are open to all

Required APIs by Screening Task

URL
/addTask/
Live Example URL

https://shivambharadwaj.com/demo/naive-todo-app-api/addTask/

Method
POST
Requirement
Accepts JSON Object

The API requires a JSON Object with attribute taskName. Other Attributes such as isDone, doneAt are optional.

Returns
JSON Object

On Successfull execution, API returns JSON Response with Attrbutes status & task.

task contains task attributes as described in problem statememt i.e. taskId, taskName, isDone, doneAt and createdAt.

On Unsuccessfull Execution, a Relevant Response is returned

URL
/editTask/< taskId >
Live Example URL

https://shivambharadwaj.com/demo/naive-todo-app-api/editTask/1

Method
POST
Requirement
Accepts JSON Object

The API requires a JSON Object with attribute taskId and taskName. Other Attributes such as isDone, doneAt are optional.

The API requires that taskId supplied in JSON obj and URL should be same.

Returns
JSON Object

On Successfull execution, API returns JSON Response with UPDATED Attrbutes status & task.

task contains task attributes as described in problem statememt i.e. taskId, taskName, isDone, doneAt and createdAt.

On Unsuccessfull Execution, a Relevant Response is returned

URL
/tasks/
Live Example URL

https://shivambharadwaj.com/demo/naive-todo-app-api/tasks/

Method
GET
Requirement
None

The API requires no input

Returns
JSON Object Array

The API returns JSON object with attributes status and tasks.

tasks attribute contains JSON obj Array with all tasks.
